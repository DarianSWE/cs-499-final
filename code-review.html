What the Original Code Does

CoursePlanner-CS300 is a C++ console application that loads computer science course records from a CSV file into a custom hash table, then provides a menu-driven interface to print a sorted course list or search for a specific course and view its prerequisites. The hash table uses separate chaining to resolve collisions, with each bucket holding a vector of Course objects.

Key Findings by Category

Software Design & Engineering
  • StructureThe entire application lives in a single file with no separation of concerns between the data model, the hash table, and the user interface.
  • DefensiveInvalid menu input causes a silent cin failure and an infinite loop. Malformed CSV rows are inserted without validation. Print and search options can run before any data is loaded.
  • IntegrityNo duplicate course ID detection exists, so re-inserting a course ID produces undefined search behavior.
Algorithms & Data Structures
  • Hash LogicThe hash function sums character ASCII values, which is not position-sensitive. Structured course IDs like CS100 and CS200 differ by a single character and cluster into neighboring buckets.
  • PerformanceClustered buckets degrade average search time from O(1) toward O(n) as the dataset grows.
Databases
  • PersistenceAll data lives in a flat CSV file re-read on every run, with no schema, no integrity constraints, and no query capability.
  • SecurityFile input is parsed and stored without sanitization, leaving the program vulnerable to malformed or malicious input.

Planned Enhancements

Each finding above was addressed by a dedicated enhancement: a modular refactor with full input validation (Category One), a replacement of the hash function with the DJB2 algorithm (Category Two), and a Python and SQLite companion script for structured, secure data persistence (Category Three). Full detail on each is available on its own page.
