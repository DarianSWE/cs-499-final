Darian Gernand

Computer Science, SNHU: transitioning into healthcare data analytics

Program Reflection & Professional Growth

Coming into the Computer Science program with a background in pharmacy, my goal was to bridge two worlds: the clinical and operational knowledge I built over years in healthcare, and the technical skills needed to work with the data that drives decisions in that space. The program gave me the structure to do that, building from the basics of application development through data structures, software testing, database management, and now this capstone. The clearest measure of growth isn’t any single course, but the cumulative ability to break a problem into components, build something that works, and critically evaluate whether it works well.

Collaboration & Process

CS-250, the Software Development Lifecycle course, introduced me to Agile methodology and the discipline of iterative development, breaking work into sprints and building in checkpoints for feedback rather than delivering everything at once. That mindset shaped how I approached this capstone directly: each milestone was scoped as its own deliverable with a specific outcome, which kept the work manageable and made instructor feedback actionable before moving to the next stage.

Communicating with Stakeholders

CS-320, Software Testing Automation and Quality Assurance, required documenting not just what code did but why testing decisions were made and what risks remained uncovered. That kind of written technical reasoning is exactly what a data analyst is expected to produce for a non-technical audience. In healthcare, the people making decisions based on data analysis are often clinicians or administrators, not engineers writing clearly about technical work in terms a stakeholder can evaluate is a skill I’ll use daily in my new role.

Data Structures, Algorithms & Software Engineering

Data structures and algorithms became a genuine interest after CS-300. Understanding how a data structure choice affects performance, and how an algorithm behaves under different input conditions, gave me a way of thinking that carries directly into analytics work, a poorly written query against a large healthcare dataset can produce the same result as a well-written one but take ten times as long. CS-340, Advanced Programming Concepts, gave me hands-on experience connecting application logic to a database backend, laying the groundwork for the database enhancement in this capstone.

Security

Security was woven throughout the program. CS-405, Secure Coding, introduced the mindset of writing code that anticipates misuse rather than just correct use, a mindset that directly shaped the input validation work in this capstone and the parameterized query design in the database enhancement.

ePortfolio Artifact Summary

The three artifacts in this ePortfolio all derive from a single C++ course planning application originally developed in CS-300. Choosing one artifact and enhancing it across all three categories was intentional: it demonstrates growth in depth rather than breadth, taking existing, working code and improving it in substantive and distinct ways rather than producing three disconnected projects.

The first enhancement refactored the application’s single-file structure into a modular, object-oriented design and added comprehensive input validation throughout. The second replaced a weak character-summation hash function with the DJB2 algorithm, fixing a real collision problem on structured course IDs. The third introduced a standalone Python companion script using SQLite, replacing flat-file CSV storage with a proper relational database backend secured by parameterized queries.

Taken together, these enhancements tell a coherent story about how I approach technical work: identify weaknesses with specificity, choose improvements grounded in established principles, implement them with attention to correctness and security, and document the reasoning clearly.
