Databases

A Python & SQLite companion script replacing flat-file CSV storage with a secure relational backend.

Outcome 4 · Innovative Techniques Outcome 5 · Security Mindset

Artifact: course_planner.py · Python 3 / sqlite3

The Artifact

This enhancement extends the same CoursePlanner project with a standalone Python script that manages the same course data using Python’s built-in sqlite3 module. In its original form, the application stored all data in a plain-text CSV file re-read on every run, with no schema, no integrity constraints, and no query capability.

Why This Enhancement

A flat CSV file works for a small class project but breaks down as a real architecture. There’s no way to enforce that a course ID is unique, no way to guarantee a title field isn’t empty, and no way to query the data without loading the entire file into memory. File input was also read and stored without any sanitization, leaving the program exposed to malformed or malicious data.

What Changed

The script defines a formal schema with a primary key and a required title field, then imports the existing CSV using parameterized queries throughout.

# course_planner.py — schema definition
cursor.execute("""
    CREATE TABLE IF NOT EXISTS Courses (
        courseID      TEXT PRIMARY KEY,
        title         TEXT NOT NULL,
        prerequisites TEXT
    )
""")

# Parameterized insert — prevents SQL injection
cursor.execute(
    "INSERT OR IGNORE INTO Courses (courseID, title, prerequisites) VALUES (?, ?, ?)",
    (course_id, title, prerequisites)
)

The ? placeholders ensure that data is always passed as a bound parameter rather than concatenated into the SQL string, which is the standard defense against SQL injection. INSERT OR IGNORE enforces the primary key constraint automatically, duplicate course IDs are skipped without crashing the program. Sorting and lookup are now handled by the database itself through ORDER BY and parameterized WHERE clauses, rather than custom logic in the application.

Course Outcomes

Outcome 4, applying well-founded and innovative techniques, is demonstrated by using Python’s standard-library sqlite3 module to deliver structured storage and querying with no external dependencies. Outcome 5, developing a security mindset, is demonstrated throughout, parameterized queries, schema-level constraints, and row validation before insertion together ensure malformed or malicious data never reaches the database.

Reflection

This enhancement clarified what a database actually provides beyond storage. The primary key constraint meant I never had to write duplicate-detection logic by hand, the database enforces it. The ORDER BY clause meant I never had to sort in application code, the database handles that too. Offloading those concerns to the database layer isn’t just cleaner code, it’s a fundamentally more reliable architecture, and one I expect to lean on heavily in healthcare analytics work.
